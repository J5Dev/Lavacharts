DataTables

The heart of the Chart, containing all of the data to display on the screen.

Data can come from anywhere:  SQL results,external APIs, manually, in a loop, however you need.


Methods

Lava::DataTable( string $label = '' )

Creates new DataTable object, and optionally stores in memory with label if assigned.

  • paramstringAssigns a label to the DataTable, and stores it in the Lava Class. (optional, but recommended)
  • ORWith no parameter, a DataTable is returned and not stored in the Lava Class
  • returnsobject (dataTable)A DataTable object.

addColumn( string $type, string $opt_label = '', string $opt_id = '')

Add columns to the DataTable.

  • paramstringType of data that the column will represent.
    Valid values include: 'string', 'number', 'boolean', 'date', 'datetime', 'timeofday'.
  • paramstringColumn label (optional).
  • paramstringColumn ID (optional).
  • returnsobject (dataTable)A DataTable object.

addRow(array $opt_cellArray = NULL)

Add a row to the DataTable

  • paramarrayArray of data points, with equal number of items to number of columns.
  • returnsobject (dataTable)A DataTable object.
  • The order of data corresponds to the order in which the columns were added
    Number of items in row, must equal number of columns. EX: count($opt_cellArray) == count(columns).
    Additionaly, if called with no parameter, then a NULL value will be added to the DataTable, skipping a row.
