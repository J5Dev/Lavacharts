Lava API

The Lava Class is used to create & store DataTables and Charts
as well as creating configuration objects and placing the chart into the page.


Methods

Lava::DataTable( string $label )
  • See DataTables

Lava::XxxxChart( string $label )
  • See Charts & Config

Lava::div( string $elementID | int $width, int $height )

Builds a div html element with ID for a chart to be rendered into.

  • paramstringAssigns a label to the DataTable, and stores it in the Lava Class.
  • ORSet the dimensions as (int)s and the element ID is assigned automatically from the outputInto() method.
  • paramintSets CSS width for the div.
  • paramintSets CSS height for the div.
  • returnshtmlDiv HTML element to place in page for the chart to be rendered into.

Lava::hasErrors()

Checks if any errors were encountered while setting up the chart.

  • returnsbooleanTRUE if errors occured, otherwise FALSE.

Lava::getErrors()

Returns any errors that may have been encountered while configuring.
Errors are wrapped with the defined html tag that can be set in the "app\config\packages\khill\lavacharts\config.php".

See how to publish the LavaChart config file.
  • returnsstringError string[s], wrapped in html element.

DataTables

The heart of the Chart, containing all of the data to display on the screen.

Data can come from anywhere:  SQL results,external APIs, manually, in a loop, however you need.


Methods

Lava::DataTable( string $label = '' )

Creates new DataTable object, and optionally stores in memory with label if assigned.

  • paramstringAssigns a label to the DataTable, and stores it in the Lava Class. (optional, but recommended)
  • ORWith no parameter, a DataTable is returned and not stored in the Lava Class
  • returnsobject (dataTable)A DataTable object.

addColumn( string $type, string $opt_label = '', string $opt_id = '')

Add columns to the DataTable.

  • paramstringType of data that the column will represent.
    Valid values include: 'string', 'number', 'boolean', 'date', 'datetime', 'timeofday'.
  • paramstringColumn label (optional).
  • paramstringColumn ID (optional).
  • returnsobject (dataTable)A DataTable object.

addRow(array $opt_cellArray = NULL)

Add a row to the DataTable

  • paramarrayArray of data points, with equal number of items to number of columns.
  • returnsobject (dataTable)A DataTable object.
  • The order of data corresponds to the order in which the columns were added
    Number of items in row, must equal number of columns. EX: count($opt_cellArray) == count(columns).
    Additionaly, if called with no parameter, then a NULL value will be added to the DataTable, skipping a row.

Charts & Configuration


LavaCharts is extensively customizable, utilizing all the configuration of Google Charts

All configuration options closely mirror the javascript structure of the Google Chart API.
The was done to make it easy for looking up references or if you have used the Chart API before.


Currently Supported Charts

  • Line,  Area,  Pie / Donut,  Column,  Geo,  More Coming Soon...

Chart Methods:

  • The Chart methods always return themselves, so options are chainable.
  • The Charts all extend the parent "Chart" class to provide common options to all specific Chart objects.
  • Charts also have specific config methods, to further customize the chart. (use the above links to jump to a section)

Setting up the options for the charts:

  • This is the list of options that all charts have in common, since they extend from the Chart class.
  • Options can be set by using the method names as keys in an associative array mapped to values, then passed into the setConfig() methodORoptions can be set by using/chaining the methods shown below.
  • The outputInto() method is an exception, and cannot be chained, since it returns the javascript blocks for inserting into views.

Parent Chart / Common Methods

Lava::XxxxChart( string $label = '' )

Replace "Xxxx" with any supported chart, no quotes and new chart of that type will be created. (note, use CamelCase)

  • paramstringSets a label for the chart to correspond with a DataTable. (optional, but recommended)
  • returnsobject (Chart)Returns a new chart of specified type for chaining.

setConfig( array $option )
  • paramarraySets configuration options from array of values.
  • returnsobject (Chart)Returns the chart for chaining.

outputInto( string $elementID )
  • elementIDstringSets the ID of the HTML element to output the chart into.
  • returnsscript blockJavascript blocks to insert into the page.

dataTable( DataTable $dataTable | string $label )

If the DataTable was created with a label that doesn't match the Chart label, then pass the label to this function.
Otherwish, if the DataTable was created with no title, then pass a DataTable object to this function to assign to the chart.

  • paramobject (DataTable)Sets the DataTable to use for the chart.
  • ORparamstringSets the labeled DataTable to use for the chart.
  • returnsobject (Chart)Returns the chart for chaining.

backgroundColor( backgroundColor $backgroundColor)
  • paramobject (backgroundColor)Sets the chart background fill color, border and border color.
  • returnsobject (Chart)Returns the chart for chaining.

chartArea( chartArea $chartArea )

Defines where the chart itself is drawn, excluding axis and legends.

  • paramobject (chartArea)An object with members to configure the placement and size of the chart area.
  • returnsobject (Chart)Returns the chart for chaining.

colors( array $colorArray )

The colors to use for the chart elements.

  • paramarrayAn array of strings, where each element is an HTML color string.
  • returnsobject (Chart)Returns the chart for chaining.

fontSize( integer $fontSize )

The default font size, in pixels, of all text in the chart. You can override this using properties for specific chart elements.

  • paramintegerNumber of pixels
  • returnsobject (Chart)Returns the chart for chaining.

fontName( string $fontName )

The default font face for all text in the chart. You can override this using properties for specific chart elements.

  • paramstringValid font name.
  • returnsobject (Chart)Returns the chart for chaining.

height( integer $height )

Height of the chart, in pixels.

  • paramintegerNumber of pixels
  • returnsobject (Chart)Returns the chart for chaining.

Config Objects

All of the configuration objects can be created by one of two ways:

  • Passing an associative array into the constructor.
  • Using the configuration options as methods of the object. (Chainable, see Chaining section)
  • The only exception is the jsDate object, it mimics the javascript date signature.

Method Chaining

PHP 5.3+ allows us to chain almost all aspects of LavaCharts

Provided are some examples of method chaining.

Configuration Objects

$titleStyle = Lava::textStyle()->color('#F3BB00')
                               ->fontName('Arial')
                               ->fontSize(20);
                
$backgroundColor = Lava::backgroundColor()->stroke('#113bc9')
                                          ->strokeWidth(4)
                                          ->fill('#ffd');

DataTables

$chartData = Lava::DataTable('Times')
                 ->addColumn('date', 'Dates', 'dates')
                 ->addColumn('number', 'Estimated Time', 'schedule')
                 ->addColumn('number', 'Actual Time', 'run')
                 ->addRow(array(Lava::jsDate(2013, 8, 1), 15, 23))
                 ->addRow(array(Lava::jsDate(2013, 8, 2), 12, 21))
                 ->addRow(array(Lava::jsDate(2013, 8, 3), 16, 20))
                 ->addRow(array(Lava::jsDate(2013, 8, 4), 13, 23))
                 ->addRow(array(Lava::jsDate(2013, 8, 5), 18, 19))

Charts

Lava::LineChart('Times')->title('Times for Deliveries')
                        ->fontSize(20)
                        ->fontName('Impact')
                        ->colors(array('blue', 'green', 'red'))
